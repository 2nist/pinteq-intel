Current state
How discovery flows today
01
Courthouse — discovery received
DVD, USB, or physical media. Whatever format the prosecution provides. No naming standard. No manifest. No record of what arrived.
Physical hand-off
↓ manual copy · no log
02
4TB external drive — first landing
Files copied as-is. Original prosecution naming intact. No case association enforced. Accumulates over time with no organization.
Raw dump
↓ manual transfer · no log
03
10TB working drive — working storage
Mixed folders. Inconsistent naming. Some cases have folders, some don't. No version control. Duplicate risk. No chain of custody.
Working copy
↓ manual upload by secretary
04
Smokeball — partial upload
Some files make it in. Naming inconsistent. Structure varies by who uploaded. Not everything on the drives is in Smokeball.
Incomplete truth
↓ manual search when needed
05
Analysis — manual and scattered
Intelligence lives in freeform memos, emails, and memory. Witness contradictions found by accident. No searchable case intelligence layer.
Unstructured
Where this breaks down
No chain of custody
No record of what discovery arrived, in what format, or when. Files go missing with no paper trail.
Files are unfindable
SUSPICIOUS_CONDITION_NOISE_SUBJECT-3 · ASSAULT · 2025-12-08_19-54-17_utc_e37 · Case-WY2534851 — four formats in one folder. Opening each file is the only way to know what's in it.
11GB of unindexed video
Body cam and surveillance files sit in Smokeball with no transcription, no searchable content, no index. The evidence inside is inaccessible without playing every file.
14 tasks overdue
No automated task generation from case events. Deadlines tracked manually across a 64-matter caseload.
With Pinteq
How discovery flows with the pipeline
01
Courthouse — intake manifest created
When discovery arrives, a 30-second intake form logs: case ID, date received, source, media type, file count. Chain of custody begins before anything is copied.
Logged on arrival
↓ copied to drop folder
02
Pinteq intake pipeline — auto-activates
Pipeline watches the drop folder. The moment files land it classifies every file by type, assigns document category, renames to the agreed convention, and logs to the master index.
Automatic
↓ background processing
03
Content extraction — what's inside
PDFs → text extracted. CSVs → structured rows. Audio/video → Whisper transcription queue. Spanish flagged for translation. All content searchable.
Background job
↓ API upload
04
Smokeball — structured and complete
Every file uploaded to the correct matter folder with the normalized name. Tags applied. Chain of custody memo created. Chuck sees organized files without touching the pipeline.
Cloud confirmed
↓ intelligence layer
05
Intelligence — structured and queryable
Witness briefs generated. Relationship maps built. Chronology reconstructed. Contradictions flagged. Everything searchable by name, date, topic, or connection.
Queryable
